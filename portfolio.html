Uitwerking & Bugs

Mijn uitwerking is uiteindelijk iets anders geworden als dat je hiernaast ziet, je klikt op een leeruitkomst en vanuit daar kun je door de buttons verder navigeren naar de projecten. Ik heb de buttons allemaal een bijpassende foto gegeven zodat je in het kort ziet waar het project over gaat. Ik heb over de gehele website animate on scroll toegvoegd, dit heb ik gedaan om de website wat meer interactief te maken en wat meer te laten bewegen.

De website is helaas niet helemaal waterdicht, zo is hij niet responsive en kunnen sommige dingen verschillende op andere schermen. Ik heb mijn portfolio gemaakt op een scherm van 1920 bij 1080 pixels. Op dit scherm staat alles dus ook het meest recht. Wanneer ik het op mijn laptop bekijk dan scale ik mijn webpagina naar 80%, op deze manier komt ook alles goed te staan.

Code

Ik heb voor de code van mijn portfolio een hele boel files aangemaakt. Heel veel html files, verschillende css files en wat javascript files. Ik streef er naar om in mijn volgende project zoals deze een betere mappen structuur te creeëren en alles wat netter te krijgen.

HTML uitleg

Mijn homepage is erg simpel, ik heb doormiddel van een list mijn navigatie balk gemaakt. Ik heb deze zo gestijld dat alles netjes onder elkaar komt te staan aan de rechterkant van de pagina. Naast de navigatie balk vertel ik ook nog kort iets over mezelf en heb ik mijn favoriete muziek in een spotify playlist gezet en deze op de homepage gezet.

Elke navigatie pagina ziet er ongeveer het zelfde uit als op de foto hier naast, ik heb gewerkt met secties om zo een soort start pagina te krijgen elke keer als je op een project klikt. Door op de muis met de pijltjes te klikken scroll je automatisch naar sectie 2. In sectie 2 staan alle buttons die naar de projecten gaan. Wanneer je op een project klikt krijg je hetzelfde idee met de secties, als je naar onder scrollt heb ik 2 verschillende classes aangemaakt. Een class voor content aan de linker kant en een voor content aan de rechter kant. Links is voor de foto's en rechts is voor tekst zoals je ziet.

CSS uitleg

Om te beginnen zijn mijn CSS files een beetje een rommeltje. In het begin gooide ik alles in 1 CSS files. Later kwam ik er achter dat dit me af en toe tegen werkte. Hierna heb ik meerdere CSS files aangemaakt, zo heb ik er een voor het recht zetten en stijlen van de tekst aan de rechter kant een een voor het recht zetten en stijlen van de foto's links.

Ik heb ook voor heel veel dingen nieuwe classes aangemaakt, ik heb geleerd dat dit niet altijd nodig is. Het wordt vaak alleen maar minder overzichtelijk en zijn er classes die al bestaan of dezelfde stijl hebben, zo krijg je dus overbodige CSS classes. Dit is iets wat ik af wil leren.

JavaScript uitleg

Ik gebruik voor mijn website een aantal kleine javascript functies om mijn portfolio wat beter te maken. Zo heb ik een custom scrollbar toegvoegd deze heb ik gestijld in de kleuren van mijn website. Naast de scrollbar maak ik ook gebruik van de library animate on scroll, dit maakt de website wat meer smooth en komt er beweging in de website. Ook heb ik boven aan de pagina een muis met pijltjes erbij staan, deze knopt dient er voor om via een smooth scroll naar sectie 2 te gaan.

Als laatste heb ik links boven in de hoek ook nog een pijl om naar de vorige pagina te gaan. Dit script onthoud waar je hiervoor was en stuurt je terug naar die pagina wanneer je op de pijl klikt. Hier zitten wel wat bugs in, zo stuur hij je bijvoorbeeld terug naar sectie 1 als je op knop geklikt hebt om naar sectie 2 te gaan.

Wat heb ik geleerd?

Tijdens het maken van mijn portfolio heb ik heel veel geleerd, ik begon met eigenlijk geen flauw benul van hoe ik moest beginnen. Ik ben toen maar wireframes gaan maken, deze heb ik later uitgewerkt in de eerste versie van mijn homepagina. Door de de tijd heen leerde ik steeds meer dingen over html en css. Als ik terug kijk naar hoe ik begon en wat ik nu al weet ben ik toch wel trots op mezelf. Natuurlijk is mijn portfolio niet perfect, maar ik ben blij met de vooruitgang die ik heb kunnen maken.

Wat doe ik de volgende keer anders?

De volgende keer dat ik zo'n project start zou ik wat gestructueerder werken, een nette mappenstructuur gebruiken en mijn CSS files niet overvol gooien maar dit scheiden in verschillende files en dus ook niet onnodig veel classes aanmaken. Naast dat wil ik ook leren om mijn website responsive te maken, dat is mijn portfolio nu niet. Het is wel handig dat je website op meerdere schermen goed bruikbaar is.
