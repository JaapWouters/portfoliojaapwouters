Feedpulse

Ik heb voor de code verschillende files aangemaakt. Een voor het script zelf, een voor de face-api, een html file om je camera op te laten zien en een mapje met de models van de face-api.

Groeps feedpulse

Doormiddel van Promise.all worden alle models van de face-api geladen. Zo is de tinyFaceDetector voor detectie van een gezicht. Het faceLandmark68Net model maakt het mogelijk om verschillende delen van je gezicht te laten zien. Dit zijn je wenkbrouwen, ogen, neus, en mond. FaceRecognitionNet wordt gebruikt om aan te tonen waar je gezicht is in beeld. En als laatste wordt het faceExperssionNet model gebruikt om verschillende emoties aan te tonen.

Om je camera in te laden wordt de functie startVideo gebruikt. In de method stream zet ik een source object voor video, dit maakt het mogelijk om de media die er op je webcam afspeelt in te laden.

Individuele feedpulse

Ik gebruik een event listener om code uit te voeren wanneer de webcam aan staat. Doormiddel van de functie setInterval wordt de code hierin meerdere keren achter elkaar uitgevoerd. Voor elk gezicht dat de camera opmerkt wordt gezichtsdetectie uitgevoerd, dit komt door een functie in de face-api genaamd "detectAllFaces". Deze detectie wordt elke 100 milliseconde uitgevoerd. Na het detecteren worden alle models van de face-api uitgevoerd. Al deze models worden in een canvas ingeladen. Dit canvas is op de juiste maat gestijld. Voor dat we alle models gaan tekenen wordt elke keer het canvas gecleard zodat de vorige detecties niet op het canvas blijven staan. Hierna worden alle models getekend in het canvas zodat je ze ook echt live op je webcam ziet verschijnen.

Ik heb het script van de face-api boven het script van de gezichtsdetectie gezet. Dit doe ik zodat de face-api eerder ingeladen wordt dan het gezichtsdetectie script. Als ik dit zou omdraaien ontstaan er errors omdat het programma eerst het script leest en daarna pas de data van de face-api ziet. Zoals je ziet in het HTML file is de video ook op de juiste manier gestijld. Maak je de resolutie groter dan wordt je detectie ook groter en minder accuraat.

Wat heb ik geleerd?

Tijdens de weken van sprint x heb in een korte tijd wat meer geleerd over javascript en api's. Mijn POC verliep daarintegen niet klakkeloos, zo zijn er een paar dingen waar ik tegen aan liep. Zo ben ik er bijvoorbeeld achter gekomen dat het prototype niet inlaad op een online website, maar lokaal wel. Dit gebeurd omdat POC blijkbaar heel fragiel is met waar het staat opgeslagen en hoe het geopend word. Een echte verklaring heb ik er niet voor. Zie het gifje boven aan de pagina voor een voorbeeld.

De verdere fouten die ik tegen kwam zie je in het PDF-bestand hieronder.

Wat doe ik de volgende keer anders?

Wat ik de volgende keer anders wil gaan doen is eerder dingen plannen. Ik ben in week 2 pas echt begonnnen met dit project voor sprint x, dit kwam omdat ik het ook erg druk had met mijn portfolio. Bijhorend het eerder plannen wil ik meer feedback vragen en dit dan ook optijd kunnen vragen.
